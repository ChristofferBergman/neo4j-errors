Neo4j Error Lookup

Earlier versions of Neo4j (<= 5.x) lacked a reliable error code, which forced applications to either rely on the less reliable Neo4j Status Code or the actual error messages instead. Relying on the Neo4j status code or the textual error message is now deprecated, and instead applications should rely on the new GQLSTATUS code. This tool will help you figure out what GQLSTATUS to use.

The data model looks like this:


Each Error node represents one error as seen in the earlier versions. It has a reference property used just as a unique identifier, this isn't seen anywhere in the software or protocol. It also has the olderror property, which is the now deprecated error message template. All errors point to a Neo4jStatus node with the Neo4j Status Code used in earlier versions (also deprecated now). Many errors point to the same Neo4j Status Code. All errors also points to a GqlStatus code with a GQLSTATUS relationship (also here many errors can point to the same GqlStatus). In some cases this code is a more generic code, and in those cases it also has a Cause relationship to a more specific code.
